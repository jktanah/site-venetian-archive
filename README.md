# site-venetian-archive

本仓库用于归档和发布多个独立 HTML 页面。这些页面分别对应不同的网站内容，不隶属于任何单个域名或具体项目。

## 项目简介

site-venetian-archive 是一个静态页面集合，旨在保存一组分散的网页资源。仓库中的每个 HTML 文件均可独立访问，不依赖特定后端服务或数据库。这些页面可能来自不同时期、不同主题，仅作存档和展示用途。

## 目录说明

```
site-venetian-archive/
├── index.html          # 仓库主页面（可选）
├── pages/              # 存放独立 HTML 页面的目录
│   ├── page1.html
│   ├── page2.html
│   └── ...
├── assets/             # 公共资源文件（CSS、图片、字体等）
│   ├── css/
│   ├── images/
│   └── ...
└── README.md           # 本文件
```

- **pages/**：所有归档的独立 HTML 页面存放于此。每个文件为一个完整页面，可直接在浏览器中打开。
- **assets/**：存放页面可能引用的公共静态资源，如样式表、图片等。资源文件按类型分文件夹整理。

## 页面归档说明

- 每个页面均为独立完整的 HTML 文件，包含内嵌或引用的样式、脚本和资源。
- 页面内容可能包含文本、图像、链接等元素，但所有链接均指向相对路径或本仓库内部资源，避免对外部服务的依赖。
- 归档目的仅为保存和展示，不承诺页面内容的时效性或完整性。部分页面可能因技术演进而存在显示差异。

## 维护说明

- 本仓库的维护工作包括：定期检查页面可访问性，更新资源引用路径，添加新的归档页面。
- 若发现页面显示异常（如资源加载失败、布局错乱），可提交 Issue 说明问题页面和具体情况。
- 欢迎提交 Pull Request 添加新的独立 HTML 页面，但请确保页面内容不违反相关法律法规和平台政策。

## 使用方式

1. 克隆本仓库到本地：
   ```bash
   git clone https://github.com/your-username/site-venetian-archive.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的 HTML 文件，或通过本地服务器（如 Python 的 `http.server`）运行查看。
